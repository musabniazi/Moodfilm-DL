🔬 How Our ML Works

Three algorithms working together to match your mood to a movie

🗣️
Sentiment Analysis (NLP)
We tokenize your search query and score each word against positive/negative lexicons, then map the tone to one of 8 moods using weighted keyword matching.
Raw text → Tokenise → Keyword score → Mood map
"I want something scary and tense"
Negative score: +2 · Horror keywords: +2
→ Detected mood: horror (confidence 0.84)
📐
Content-Based Filtering
Each movie and each mood is encoded as a binary genre vector. We compute cosine similarity between them — the closer the angle, the stronger the match.
Action Romance Sci-Fi Horror
Your mood 1 0 1 0
Movie A 1 0 1 0
Movie B 0 1 0 1
Movie A cosine similarity: 1.00 · Movie B: 0.00
🗂️
Mood Clustering (k=4)
Movies in a result set are soft-clustered into 4 mood groups (Uplifting, Dark, Adventure, Thoughtful) by computing genre-vector distance to each cluster centroid.

⚔️ LSTM vs Traditional ML

How deep learning sequence modeling compares to our classical ML algorithms

Capability 🔵 Traditional ML 🟣 LSTM Deep Learning
Mood Detection Sentiment Analysis — keyword scoring on single input ✓ Sequence Prediction — learns from mood history
Pattern Recognition K-Means Clustering — groups by static genre vectors ✓ Temporal Pattern Recognition — detects evolving patterns
Prediction Type Static — same text always gives same mood ✓ Dynamic — predictions adapt to your mood sequence
Memory No memory — each query is independent ✓ Gated memory — forget, input, output cell states
Recommendation Logic Cosine Similarity — fixed genre-vector distance ✓ Sequence-aware — accounts for where you're heading
Inference Speed ✓ <5ms — pure keyword lookup ~12ms — matrix operations over sequence
Training Required ✓ No training — hardcoded lexicons Pre-defined affinity weights (simulated training)
